# FRAGA DASHBOARD - WORKFLOW DEVOPS COM IA

## 🏗️ ARQUITETURA DE TRABALHO EM EQUIPE

### PAPEL: DevOps Lead (Você) 
- ✅ Decisions estratégicas
- ✅ Priorização de tarefas
- ✅ Code review de mudanças críticas
- ✅ Validação de deploys
- ✅ Monitoramento de prod

### PAPEL: DeepSeek (Executor Automático)
- ⚙️ Executar comandos diretos
- ⚙️ Implementar features (follow specs)
- ⚙️ Bugfixes automáticos
- ⚙️ Testes e validações
- ⚙️ Logs e documentação

---

## 📋 PROTOCOLO DE TRABALHO

### 1️⃣ VOCÊ DEFINE (Strategy & Direction)
```
Exemplo:
"DeepSeek, refactor da página de dashboard:
- Componente Card deve ter loading state
- Implementar cache com React Query
- Adicionar testes unitários
- Commit message: 'refactor(dashboard): add loading states'"
```

### 2️⃣ DEEPSEEK EXECUTA (Automated Execution)
```
- Analisa requirements
- Faz o código
- Roda testes
- Faz commit/push
- Reporta status com logs
```

### 3️⃣ VOCÊ VALIDA (QA & Approval)
```
- Review resultado
- Testa em staging
- Aprova ou pede ajustes
- Deploy em prod
```

---

## ⚡ PADRÕES DE COMANDOS RÁPIDOS

### Para Você dar (como chefe):
- "DeepSeek, status" → relatório completo
- "DeepSeek, fix [bug description]" → bugfix automático
- "DeepSeek, feature [requirement]" → feature completa
- "DeepSeek, deploy prod" → deploy com validação
- "DeepSeek, urgent fix" → prioridade máxima

### Resposta padrão do DeepSeek:
```
✅ INICIADO: [tarefa]
🔄 PROGRESSO: [passo 1] → [passo 2] → [passo 3]
📊 RESULTADO: [sucesso/erro com detalhes]
📋 LOGS: [relevantes]
➡️ PRÓXIMO: [aguardando sua validação]
```

## 🛠️ TECH STACK ATUAL
- **Frontend:** React 18 + TypeScript + Vite
- **Backend:** Node.js ESM + DrizzleORM
- **Database:** TiDB (MySQL compatible)
- **Infra:** PM2 + Ubuntu
- **Build:** npm run build
- **Deploy:** pm2 restart

---

## 📁 ESTRUTURA DE BRANCHES
- `main` → produção (você aprova)
- `staging` → testes (DeepSeek faz)
- `feature/*` → features (DeepSeek executa)
- `hotfix/*` → bugs críticos (DeepSeek urgent)
